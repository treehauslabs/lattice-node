# Detailed Layer Specifications

These files contain exhaustive implementation details for each layer of the
Lattice blockchain stack. They were generated by reading every source file
in the five repos.

| File | Layer | Contents |
|------|-------|----------|
| SPEC.md (root) | All | Condensed overview + test harness |
| cashew.md | cashew | CID computation, type hierarchy, RadixTrie, resolution, storage |
| ivy.md | Ivy | Wire protocol (57 message types), DHT, Volume protocol, tally, pins |
| volumebroker.md | VolumeBroker | SQLite schema, broker chain, BrokerStorer, pin lifecycle |
| lattice.md | Lattice | Block structure, ChainState, validation pipeline, cross-chain swaps |
| lattice-node.md | lattice-node | Block processing, sync, mining, mempool, RPC API, bootstrap |

## Using with an agent

Point the agent at SPEC.md first for the overview and test harness, then
at the specific layer spec when implementing that layer. The smoke tests
in `SmokeTests/` are the acceptance criteria — pass them all unchanged.
